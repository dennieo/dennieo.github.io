# Нестандартные подходы к автоматическому трейдингу

*Версия 1.0 — дополнение к [ARCHITECTURE.md](ARCHITECTURE.md)*

Классические индикаторные стратегии (v1: EMA+RSI) торгуют против тех же
индикаторов у миллионов других участников. Ниже — пять альтернативных источников
преимущества, их честная оценка и прототип выбранного подхода.

---

## 1. Smart Money Tracking — следование за кошельками успешных трейдеров ✅ *реализовано*

### Идея

Блокчейн прозрачен: сделки трейдеров на DEX публичны и привязаны к адресам
кошельков. Значит, можно **найти кошельки со стабильно прибыльной историей и
использовать их действия как сигнал** — по сути, автоматический copy-trading без
разрешения того, кого копируешь. Это анализ открытых данных, полностью легально.

### Почему Hyperliquid

Проблема Ethereum/Uniswap-подхода — восстанавливать PnL кошелька из сырых свопов
дорого и неточно (нужны цены на момент каждого свопа, учёт газа, аирдропов).
**Hyperliquid** — крупнейшая деривативная DEX — решает это за нас:

- публичный API **без ключей и без оплаты** (`POST https://api.hyperliquid.xyz/info`);
- по любому адресу доступны **все сделки** (`userFills`) с уже посчитанным
  `closedPnl` по каждому закрытию — не нужно ничего реконструировать;
- **текущие открытые позиции** любого адреса (`clearinghouseState`) — видно,
  кто сейчас в лонге и с каким входом;
- публичный лидерборд трейдеров по PnL — готовый источник кандидатов.

### Конвейер

```
 Discovery                Scoring                  Tracking                Signal/Execution
┌───────────────┐   ┌──────────────────┐   ┌────────────────────┐   ┌──────────────────────┐
│ лидерборд /   │──▶│ userFills:       │──▶│ каждые 60с опрос   │──▶│ консенсус: ≥K из     │
│ адреса из     │   │ pnl, winrate,    │   │ позиций топ-N      │   │ топ-N в лонге по     │
│ конфига       │   │ profit factor,   │   │ кошельков; события │   │ монете → BUY-сигнал; │
│               │   │ число сделок →   │   │ opened/closed long │   │ консенсус распался → │
│               │   │ smart-score      │   │                    │   │ SELL. Telegram и/или │
└───────────────┘   └──────────────────┘   └────────────────────┘   │ спот-ордер Binance   │
                                                                    └──────────────────────┘
```

1. **Discovery.** Берём кандидатов из лидерборда (или задаём адреса вручную в
   конфиге — например, найденных через обозреватели вроде Hypurrscan).
2. **Scoring.** По каждому кошельку скачиваем историю сделок и считаем:
   суммарный PnL, winrate, profit factor, число закрытий. Отсекаем кошельки
   с < 30 сделок (мало статистики — может быть просто везение) и с отрицательным
   PnL. Остальным присваиваем smart-score, берём топ-N.
3. **Tracking.** Раз в минуту опрашиваем открытые позиции топ-кошельков и
   фиксируем события: «кошелёк X открыл лонг BTC», «кошелёк Y вышел».
4. **Consensus → сигнал.** Один кошелёк может ошибаться или хеджироваться.
   Сигнал BUY — когда **не менее K из N** (по умолчанию 3) умных кошельков
   одновременно в лонге по монете; SELL — когда консенсус распался.
5. **Execution.** По умолчанию — только уведомления в Telegram (режим
   наблюдения). Опционально (`auto_trade: true`) сигнал транслируется в спотовую
   покупку на Binance через **тот же Executor и RiskManager**, что и в v1: тот же
   1% риска, те же стопы по ATR, тот же kill switch.

### Риски и ограничения (честно)

| Риск | Что делаем |
|---|---|
| **Survivorship bias**: топ лидерборда — это и везунчики; прошлый PnL ≠ будущий | фильтр по числу сделок и profit factor, а не только по PnL; консенсус нескольких кошельков |
| **Запаздывание**: мы входим позже кита, выходим позже | торгуем только ликвидные монеты (BTC/ETH/SOL), где проскальзывание от задержки в минуты мало |
| **Кит может хеджировать**: его лонг на Hyperliquid — часть сложной позиции, которую мы не видим | консенсус ≥K независимых кошельков вместо копирования одного |
| **Wash-trading / манипуляция лидербордом** | скоринг по своей истории fills, а не по цифрам лидерборда; ручной whitelist адресов |
| **Перп ≠ спот**: киты торгуют с плечом и в шорт | копируем только лонги, только на споте, со своим риск-менеджментом |

### Что это даёт по сравнению с v1

Индикатор — производная от цены, т.е. от прошлого. Позиции умных кошельков —
информация о **чужих ожиданиях будущего**, которой нет в свече. Комбинация
«консенсус китов + трендовый фильтр v1» — самый интересный режим, и он
**реализован**: smart-money бот публикует консенсус в `data/consensus.json`,
трендовый бот при `smartmoney_filter.enabled: true` входит только при поддержке
умных денег (fail-open: без свежего консенсуса фильтр мягко отключается).

## 2. Funding Rate Harvesting (кэрри / дельта-нейтрал) — *монитор реализован*

Перпетуальные фьючерсы платят funding каждые 8 часов: когда рынок перегрет
лонгами, лонги платят шортам. Стратегия: **лонг спот + шорт перп того же
размера** = нулевой риск направления, доход — funding (исторически 5–20% годовых
на BTC в спокойные периоды). Это самый близкий к «безрисковому» доход в крипте,
но: нужны фьючерсы (второй счёт, маржа), доход мал на балансе в сотни долларов,
и есть риск разрыва ног при резких движениях. **Вердикт: отличный этап 2, когда
баланс вырастет до тысяч.** Реализован монитор (`python -m monitor.funding`):
оповещает в Telegram, когда годовая ставка превышает порог — чтобы не пропустить
момент, когда кэрри становится привлекательным.

## 3. Сентимент-анализ новостей через LLM

Раз в N минут собирать заголовки (CoinDesk RSS, Fear&Greed Index), прогонять
через LLM-классификатор («bullish/bearish/noise для BTC?») и использовать как
фильтр для v1: не покупать на панических новостях. **Вердикт: дешёвый фильтр,
но не самостоятельная стратегия — сигнал слишком шумный и уже в цене.**

## 4. Торговля боковика (mean-reversion / grid) — *реализовано*

Крипта ~70% времени в боковике, где трендовая стратегия молчит или пилится.
Реализован авторежим (`strategy.mode: auto`): по ADX бот определяет фазу рынка —
в тренде работает EMA-кроссовер, в боковике включается mean-reversion (покупка
разворота RSI из перепроданности, выход при возврате к середине). Каждая сделка
помечена стратегией — в отчёте видно, какая из них зарабатывает. Классическая
сетка лимитных ордеров — возможное развитие этого же режима.

## 5. Охота на каскады ликвидаций

Биржи публикуют поток ликвидаций (WebSocket `forceOrder` на Binance Futures).
Каскад ликвидаций → цена прокалывается ниже справедливой на минуты → покупка
в прокол с быстрым выходом. Работает, но требует WebSocket-инфраструктуры,
скорости и железных нервов. **Вердикт: этап 3, после стабильной работы v1/v2.**

## Что реализовано в этом репозитории

Прототип №1 — **`smartmoney/`**: клиент Hyperliquid API, скоринг кошельков,
трекер позиций с консенсус-сигналами, standalone-раннер с Telegram-уведомлениями
и опциональным автоисполнением на Binance spot через инфраструктуру v1
(paper/testnet/live — те же три режима). Запуск: `python -m smartmoney.main`.

Рекомендуемый путь: 1–2 недели в режиме наблюдения (только уведомления) →
сверить сигналы с рынком → включить `auto_trade` в `paper` → дальше по этапам v1.
